# AI Tools Discovery Workflows

English | [简体中文](README.zh-CN.md)

Discovery workflows for finding, shortlisting, testing, and monitoring AI tools across categories and use cases.

This repository is a neutral, practical resource for AI tools discovery, shortlist creation, category browsing, testing workflow, and tool monitoring. It is designed to help teams make clearer decisions, write better review notes, and avoid vague tool or workflow choices.

## Why This Repository Exists

Finding an AI tool is not the same as choosing one. A discovery workflow helps users move from broad category browsing to shortlist, test plan, evaluation notes, and monitoring for better alternatives.

## Who This Is For

- teams building an AI tool stack
- operators tracking new AI products
- users comparing tools across categories

## Core Workflow

1. Define the decision or review goal
2. Collect the required product, content, or workflow inputs
3. Use one template to make the evaluation comparable
4. Write fit, limits, and next-action notes
5. Refresh the document when products or workflows change

## Suggested Use

Start with the guide that matches your current decision point. Then copy one template into your own workspace, fill it in for one real project or product, and keep the notes for future comparison.
